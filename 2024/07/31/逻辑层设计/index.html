逻辑层设计

逻辑层概述

Session 层只用来接收发送数据，逻辑层才是对接收到的数据进行处理

更改消息节点

使用 TLV 格式，消息 id 与消息长度作为消息头，消息内容作为消息体

代码实现逻辑

  1. 逻辑层使用单例模式架构
  2. Session 层接受完客户端发来的数据后，将数据和 Session 发送给逻辑队列
  3. 逻辑层单独启用工作线程（消费者），当逻辑队列有内容时就唤醒并进行处理
  4. 析构逻辑层时，要检查逻辑队列中是否有没处理完的数据，等待工作线程确认完后再析构

单例模式模板类

逻辑层 LogicSystem

逻辑层在创建时，需要注册消息 id 对应的处理函数（回调函数）；同时创建工作线程；工作线程不断轮询处理数据；没数据就阻塞

优雅退出

C语言风格退出

  1. main 函数中创建一个新线程用来运行服务器
  2. 主线程挂起，并等待捕获退出信号
  3. 退出信号触发后，唤醒主线程，停止服务器的子线程
